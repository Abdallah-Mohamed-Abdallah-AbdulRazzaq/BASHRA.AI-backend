# 05 - User AI Dermatology APIs

This document describes all AI Dermatology APIs used by normal application users/patients.

The AI module is intentionally separated from the normal application chat. AI sessions, messages, files, results, usage counters, and shares are stored in dedicated AI tables.

---

## Base URL

```http
http://localhost:3006/api/ai-dermatology
```

In production, replace `http://localhost:3006` with the production API host.

---

## Authentication

All user APIs require a normal user JWT.

```http
Authorization: Bearer USER_ACCESS_TOKEN
```

If the token is missing or invalid, the request should return an auth error.

---

## Important Concepts

### AI session

An AI session represents one dermatology AI conversation or case. A session can contain:

- text messages
- image analyses
- document/report analyses
- final summary
- shared access with doctors
- doctor reviews on AI results

### AI result

Each AI response creates a structured result in `ai_analysis_results`.

Result types:

```text
chat_response
image_analysis
document_analysis
final_summary
```

### Secure file URL

AI medical files must be displayed through secure APIs, not public static URLs.

Use:

```http
/api/ai-dermatology/files/:fileUuid
```

Do not use:

```http
/upload/files/medical-image/...
```

### Patient consent

AI sessions require patient consent when `AI_REQUIRE_PATIENT_CONSENT=true`.

# 1. Create AI Session

Create a new AI dermatology session.

```http
POST /api/ai-dermatology/sessions
```

## Auth

User token required.

## Headers

```http
Authorization: Bearer USER_ACCESS_TOKEN
Content-Type: application/json
```

## Body

```json
{
  "title": "تحليل حالة جلدية",
  "input_mode": "chat",
  "language_code": "ar",
  "patient_consent": true
}
```

## Body fields

| Field | Type | Required | Notes |
|---|---:|---:|---|
| `title` | string | no | Session title. Default can be generated by backend. |
| `input_mode` | enum | no | `chat`, `image`, `document`, `mixed`. Usually starts as `chat`. |
| `language_code` | string | no | Usually `ar`. |
| `patient_consent` | boolean | yes when consent required | Must be true if consent is required. |

## Success response

```json
{
  "success": true,
  "message_ar": "تم إنشاء جلسة الذكاء الاصطناعي بنجاح",
  "message_en": "AI session created successfully",
  "data": {
    "id": 2,
    "uuid": "daf5bb5f-965f-4187-af0b-fad305df8239",
    "user_id": 1,
    "title": "تحليل حالة جلدية",
    "status": "active",
    "input_mode": "chat",
    "specialty": "dermatology",
    "language_code": "ar",
    "patient_consent": 1,
    "risk_level": "low",
    "ai_provider": "openai",
    "ai_model": "gpt-4.1-mini"
  }
}
```

## Common errors

| Status | Reason |
|---:|---|
| 400 | patient consent missing |
| 401 | missing or invalid user token |

# 3. Get AI Session Details

Get a full AI session with messages, files, results, latest result, and stats.

```http
GET /api/ai-dermatology/sessions/:uuid
```

## Auth

User token required.

## Example

```http
GET /api/ai-dermatology/sessions/daf5bb5f-965f-4187-af0b-fad305df8239
```

## Success response

```json
{
  "success": true,
  "message_ar": "تم جلب جلسة الذكاء الاصطناعي بنجاح",
  "message_en": "AI session retrieved successfully",
  "data": {
    "session": {},
    "messages": [],
    "files": [],
    "results": [],
    "latest_result": {},
    "stats": {
      "messages_count": 2,
      "files_count": 0,
      "results_count": 1,
      "chat_messages_count": 2,
      "image_messages_count": 0,
      "document_messages_count": 0,
      "image_files_count": 0,
      "document_files_count": 0
    }
  }
}
```

## Important returned fields

### `messages[]`

Each message has:

```json
{
  "uuid": "...",
  "sender_type": "user",
  "message_type": "text",
  "content": "...",
  "structured_content": null,
  "file": null,
  "tokens": {
    "prompt_tokens": null,
    "completion_tokens": null,
    "total_tokens": null
  }
}
```

For image/document messages, `file` contains:

```json
{
  "uuid": "FILE_UUID",
  "file_role": "skin_image",
  "file_category": "medical_image",
  "secure_file_url": "/api/ai-dermatology/files/FILE_UUID"
}
```

### `results[]`

Each result includes AI structured output and doctor review:

```json
{
  "uuid": "RESULT_UUID",
  "result_type": "chat_response",
  "severity": "mild",
  "recommended_next_step": "book_dermatologist",
  "confidence_level": "medium",
  "doctor_review": {
    "doctor_reviewed": true,
    "doctor_agreement": "partially_agree",
    "reviewed_by_doctor_id": 1,
    "doctor_notes": "...",
    "reviewed_at": "..."
  }
}
```

## Common errors

| Status | Reason |
|---:|---|
| 404 | session not found or not owned by user |
| 401 | missing/invalid user token |

# 5. Analyze Skin Image

Upload a skin image and analyze it with AI.

```http
POST /api/ai-dermatology/sessions/:uuid/images
```

## Auth

User token required.

## Body type

`multipart/form-data`

| Field | Type | Required | Notes |
|---|---:|---:|---|
| `image` | File | yes | Skin image. |
| `description` | text | no | Additional user description. |

## Example

```text
image: fpj98z6.png
description: عندي بقعة حمراء في الرقبة مع حكة منذ أسبوع وتزيد مع العرق
```

## Success response

```json
{
  "success": true,
  "message_ar": "تم رفع الصورة وتحليلها بنجاح",
  "message_en": "Image uploaded and analyzed successfully",
  "data": {
    "uploaded_file": {
      "id": 2,
      "uuid": "e2ed9336-efa1-4937-ab0b-f2cc49544629",
      "file_url": "http://localhost:3006/upload/files/medical-image/...",
      "secure_file_url": "/api/ai-dermatology/files/e2ed9336-efa1-4937-ab0b-f2cc49544629",
      "original_filename": "fpj98z6.png",
      "mime_type": "image/png",
      "file_size": 2222457,
      "file_category": "medical_image"
    },
    "result": {
      "result_type": "image_analysis",
      "severity": "mild"
    }
  }
}
```

## Mobile app rule

Use `secure_file_url` to show the image.

Do not rely on `file_url` for AI medical images.

## Usage effect

Increments:

```text
total_requests
image_analyses_count
tokens_used
```

## Common errors

| Status | Reason |
|---:|---|
| 400 | no image uploaded |
| 400 | invalid file type |
| 400 | image too large |
| 400 | max files per session reached |
| 404 | session not found or inactive |
| 429 | image/monthly usage limit exceeded |

# 7. Complete AI Session

Generate final summary and mark session as completed.

```http
POST /api/ai-dermatology/sessions/:uuid/complete
```

## Auth

User token required.

## Body

No body required.

## Success response

```json
{
  "success": true,
  "message_ar": "تم إنهاء جلسة الذكاء الاصطناعي وإنشاء الملخص النهائي بنجاح",
  "message_en": "AI session completed and final summary generated successfully",
  "data": {
    "session_uuid": "...",
    "status": "completed",
    "result": {
      "uuid": "...",
      "result_type": "final_summary",
      "severity": "moderate",
      "recommended_next_step": "book_dermatologist",
      "confidence_level": "medium"
    },
    "source_context": {
      "messages_count": 12,
      "files_count": 3,
      "results_count": 6
    }
  }
}
```

## Database effect

- Inserts AI message with final summary.
- Inserts `ai_analysis_results.result_type = final_summary`.
- Updates `ai_sessions.status = completed`.
- Saves final AI JSON in `ai_sessions.summary_json`.
- Logs provider call as `request_type = summary`.
- Records usage event `event_type = final_summary`.

## Common errors

| Status | Reason |
|---:|---|
| 400 | session already completed |
| 400 | no AI results found to summarize |
| 404 | session not found |
| 429 | usage limit exceeded |

# 8. Secure AI File Access For User

Display a private AI file owned by the authenticated user.

```http
GET /api/ai-dermatology/files/:fileUuid
```

## Auth

User token required.

## Example

```http
GET /api/ai-dermatology/files/e2ed9336-efa1-4937-ab0b-f2cc49544629
```

## Success response

Returns the actual file binary with headers:

```http
Content-Type: image/png
Content-Disposition: inline; filename="fpj98z6.png"
Cache-Control: private, no-store, max-age=0
X-Content-Type-Options: nosniff
```

## Database effect

Increments file access fields:

```text
files.access_count
files.last_accessed_at
```

## Common errors

| Status | Reason |
|---:|---|
| 403 | invalid file path or category |
| 404 | file not found or not owned by user |
| 404 | file missing on disk |

# 10. Share AI Session With Doctor

Share a session or latest result with a doctor or appointment.

```http
POST /api/ai-dermatology/sessions/:uuid/share
```

## Auth

User token required.

## Body - share with doctor

```json
{
  "doctor_id": 1
}
```

## Body - share with appointment

```json
{
  "appointment_id": 10
}
```

## Optional result selection

By default, the latest shareable result is used, preferring `final_summary` when present.

To share a specific result:

```json
{
  "doctor_id": 1,
  "result_uuid": "eca49b1d-bdf8-4a66-8f0b-c68f8a67e9e2"
}
```

## Success response

```json
{
  "success": true,
  "message_ar": "تمت مشاركة جلسة الذكاء الاصطناعي بنجاح",
  "message_en": "AI session shared successfully",
  "data": {
    "already_shared": false,
    "share": {
      "id": 1,
      "uuid": "f7161635-845d-4044-920d-c0899f743bd0",
      "share_status": "active"
    },
    "session": {
      "uuid": "daf5bb5f-965f-4187-af0b-fad305df8239",
      "status": "active"
    },
    "result": {
      "uuid": "eca49b1d-bdf8-4a66-8f0b-c68f8a67e9e2",
      "result_type": "chat_response",
      "severity": "mild"
    },
    "doctor": {
      "id": 1,
      "uuid": "1d3716ab-9507-4c75-b1d1-e4040830abb4",
      "email": "doctor@example.com",
      "full_name": "KATCH"
    },
    "appointment": null
  }
}
```

## Already shared behavior

If an active share already exists for the same doctor/session/appointment, the API returns `200` and:

```json
{
  "already_shared": true
}
```

## Common errors

| Status | Reason |
|---:|---|
| 400 | neither doctor_id nor appointment_id sent |
| 400 | no shareable AI result in session |
| 404 | doctor not found |
| 404 | appointment not found or does not belong to user |
| 404 | session not found |

# Recommended Mobile Flow

## Basic text-only flow

1. `POST /sessions`
2. `POST /sessions/:uuid/messages`
3. `GET /sessions/:uuid`
4. `GET /usage`

## Image flow

1. `POST /sessions`
2. `POST /sessions/:uuid/images`
3. Use `secure_file_url` to show image.
4. `GET /sessions/:uuid`

## Doctor sharing flow

1. User creates or uses existing AI session.
2. User sends message/image/document.
3. User shares session: `POST /sessions/:uuid/share`.
4. Doctor reviews result.
5. User calls `GET /sessions/:uuid` and sees `doctor_review`.

---

# Notes For Frontend

- Always store `session_uuid`.
- Use `result.uuid` when linking to doctor review status.
- Use `secure_file_url`, not public `file_url`.
- Treat AI responses as assistive, not final diagnosis.
- Show medical disclaimer in the UI.
- Completed sessions are read-only.
- Revoked shares should disappear from doctor access.
